    CREATE TABLE contacts (
        id INTEGER PRIMARY KEY,
        name TEXT NOT NULL,
        phone TEXT NOT NULL UNIQUE CHECK (length(phone) >= 10))

Beillesztés
    INSERT INTO table (col1, col2) VALUES (?, ?)

    INSERT INTO persons (name, age) VALUES (?, ?), (?, ?), (?, ?)
        "arthur" 33 "zaphood" 125 "marvin" 19
        ; Több sor egyidejű beillesztése.


Módosítás
    UDPATE table SET col1=val1, col2=val2 WHERE col2=?


WHERE; operátorok: =, <> v. !=, <, >, <=, >=
    WHERE col1 = 100                                    ; '(col = 100)
    WHERE col2 IN (1,2,3)                               ; `(col2 in (:vals 1 2 3))
    WHERE col3 LIKE 'An%'                               ; '(col3 like "An%")
    WHERE col4 BETWEEN 10 AND 20                        ; '(col4 between 10 and 20)
    WHERE (col5 = 'A' AND NOT col6 = 'b') OR col7 = 'c' ; '((col5 = "A" and not col6 = "b")
                                                        ;   or col7="C")
    WHERE EXISTS (SELECT 1 FROM invoices WHERE customer_id = c.customer_id)
        ; Azokat a rekordokat adja vissza, akiknek      ; `(exists (select 1 from invoices
        ; a customer_id-ja szerepel az invoices-ban     ;    ,(col 'cu-id) = ,(col 'cu-id 'c)))
    ; Joker karakterek:   '%': 0 vagy több karakter;   '_': 1 karakter


Lekérdezés
    SELECT col1, col2 FROM table WHERE col3=?

    SELECT t1c2, t1c3, t2c2, t3c2
        FROM table1
        LEFT JOIN table2 ON table1.t1c4 = table2.t2c1
        LEFT JOIN table3 ON table1.t1c5 = table3.t3c1

    SELECT DISTINCT col1 FROM table ORDER BY col2

    SELECT albumid, COUNT(trackid) FROM tracks GROUP BY albumid ORDER BY albumid
        ; Trackek száma albumonként, albumid szerinti sorrendben. A számolt
        ; oszlop neve "COUNT(trackid)" lesz.

    SELECT tracks.albumid, title, COUNT(trackid) NumberOfTracks
        FROM tracks
        INNER JOIN albums ON albums.albumid = tracks.albumid
        GROUP BY tracks.albumid
        ; Az album címét az albums táblából veszi. A számított oszlop neve 
        ; "NumberOfTracks" lesz.

    SELECT MediaTypeId, GenreId, COUNT(TrackId)
        FROM tracks
        GROUP BY MediaTypeId, GenreId
        ; Futtatott csoportosítás: (1,1), (1,2), (1,3), ..., (2,1), (2,2), ...

    SELECT col1, col2 FROM table
        ORDER BY col1 ASC, col2 DESC                    ; '((:@vals (col1 asc) (col2 desc))

Egy oszlopnak adhatunk a deklarálttól eltérő típusú értéket
- dinamikus típusrendszer.

A típusokat tárolási osztályoknak nevezzük.

NULL     Ismeretlen/hiányzó      NULL '' nélkül
INTEGER  +- egész 1-8 byte       ha nincs '', tizedespont v exp
REAL     8-byte lebegőpontos     ha nincs ' de van tizedespont v exp
TEXT     Korlátlan szöveg        ha van '' v. ""
BLOB     Korlátlan bináris obj   X'...' előtag

Kevert típusú oszlop rendezésére vannak szabályok:
https://www.sqlitetutorial.net/sqlite-data-types/    3/5



Elsődleges kulcs: a táblázat sorainak egyediségét azonosító
oszlop(csoport). Minden tábla automatikusan kap egy 64-bites
előjeles egész típusú automatikusan növekvő ROWID oszlopot, ami
egyedi azonosítóként szolgál. Ha megadunk egy integer típusú,
nem ereszkedő irányú elsődleges kulcs-oszlopot, az ennek lesz
az aliasa (már típusokkal nem!). Ha több oszlopot akarunk
együttesen elsődleges kulcsként használni, azt külön clause-ban
kell megadni táblaszinten, nem az oszlop típusánál.



NOT NULL: az oszlop nem fogad el NULL értéket. SQL szabvány
szerint az elsődleges kulcs scak ilyen lehet, de valami korai
hiba miatt SQLite-ban megengedett ha WITHOUT ROWID.
